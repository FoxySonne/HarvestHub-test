# HarvestHub

Статический одностраничный помощник для Tiles Survive. Страницы загружаются динамически, данные пользователя сохраняются локально и при наличии аккаунта синхронизируются через Supabase.

## Структура

- `components/` — общие HTML-блоки интерфейса.
- `css/core/` — базовые переменные, раскладка и адаптивность.
- `css/ui/` — общие формы, кнопки, карточки и подсказки.
- `css/features/` — стили отдельных функций.
- `data/` — игровые справочники и таблицы данных.
- `js/core/` — загрузка приложения, навигация и локальное хранение.
- `js/account/` — аккаунт и игровые профили.
- `js/sync/` — синхронизация состояния.
- `js/pages/` — контроллеры обычных страниц.
- `js/calculators/` — входные модули калькуляторов.
- `js/ipk/`, `js/turbo-vs/`, `js/troop-training/` — внутренние модули крупных калькуляторов.
- `js/season/` — сезонные расчёты.
- `js/alliance/` — API, настройки и отображение союзного штаба.
- `pages/` — HTML-содержимое динамических страниц.
- `supabase/` — SQL-схема и миграции.

## Локальный запуск

Из корня проекта:

```bash
python3 -m http.server 8000
```

Затем открыть `http://localhost:8000`.

## Проверка проекта

```bash
npm run check
```

Команда проверяет синтаксис JavaScript, относительные импорты, CSS-импорты и локальные ресурсы HTML.

## Добавление страницы

1. Добавить HTML в подходящий раздел `pages/`.
2. Если странице нужен JavaScript, создать входной модуль и экспортировать `init()`.
3. Зарегистрировать путь страницы и модуля в `js/core/navigation.js`.
4. Добавить функциональные стили в `css/features/` и подключить их через `css/style.css`.
5. Выполнить `npm run check` и проверить повторное открытие страницы через навигацию сайта.

Игровые справочники остаются в `data/`; файл `data/database.js` не следует дробить без отдельной задачи по изменению формата данных.
